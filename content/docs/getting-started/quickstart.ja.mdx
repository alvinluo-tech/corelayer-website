このガイドでは、CoreLayerをAIモデルとMCPサーバーで動作させるための最小セットアップ手順を説明します。

## ステップ1: CoreLayerを起動

CoreLayerデスクトップアプリを開きます。初期セットアップウィザードを含むコントロールセンターが表示されます。

## ステップ2: モデルプロバイダーを設定

**設定 → モデル**に移動し、プロバイダーを追加:

```text
プロバイダー: Groq（高速なローカルライクな体験におすすめ）
APIキー: your-groq-api-key
モデル: llama-3.3-70b-versatile
```

以下も使用できます:
- **MiMo** — OpenRouter経由のXiaomiのモデル
- **OpenRouter** — 複数のプロバイダーにアクセス
- **Ollama** — 完全にローカルなモデル
- **OpenAI互換** — OpenAI互換APIを持つ任意のプロバイダー

## ステップ3: MCPサーバーを接続

**設定 → MCP**に移動し、サーバーを追加:

```json
{
  "name": "filesystem",
  "command": "npx",
  "args": ["-y", "@modelcontextprotocol/server-filesystem", "/path/to/your/workspace"]
}
```

これにより、JarvisがMCPプロトコルを通じてワークスペースのファイルの読み書きにアクセスできるようになります。

## ステップ4: テスト

**Alt+Space**でコマンドパレットを開き、入力:

```text
ワークスペースのファイルを一覧表示して
```

Jarvisは以下を行います:
1. 意図を解釈
2. モデルゲートウェイを通じてリクエストをルーティング
3. ツールレジストリからファイルシステムツールを解決
4. MCPツール呼び出しを実行
5. 結果を返却

ツールに書き込みアクセスが必要な場合、パーミッションガードが一時停止して承認を求めます。

## 何が起きたのか？

```text
あなたのリクエスト
  → Jarvisが意図を解釈
  → モデルゲートウェイがGroq/llama-3.3を選択
  → ツールレジストリがファイルシステムMCPツールを解決
  → パーミッションガードがリスクレベルをチェック
  → MCPクライアントがツール呼び出しを実行
  → 結果があなたに返却
```
