---
title: 什麼是 CoreLayer？
description: 為你的桌面應用程式、工具、模型和 MCP 工作流程打造的本地優先 AI 控制層。
---

CoreLayer 是一個本地優先的桌面端 **Agent OS（AI 代理操作系統執行層）**，旨在管理 Agent、本地工具、執行狀態以及個人工作空間。

內建的 AI 助手人格被稱為 **Jarvis**。它不是又一個套殼聊天視窗，也不是單純的 IDE 插件。它透過明確的權限規則、本地沙箱會話和可驗證的執行軌跡，在你的個人應用程式、操作系統、大模型和 MCP 伺服器之間建立起安全的控制平面。

---

## 核心願景 (Product Vision)

### 1. 從「聊天歷史」過渡到「執行系統」
目前的 AI 產品仍將大部分工作堆積在單一的聊天執行緒中，隨著上下文的增長，任務狀態難以追蹤。CoreLayer 把任務當成一個可管理的執行系統：
- 你向 **Jarvis** 提出一個高層目標 (Goal)。
- 系統將其轉化為包含任務圖（Task Graph）、Agent Run（單次執行實例）、審批節點、稽核日誌和最終產物（Artifacts）的**個人工作空間 (Workspace)**。

```text
目標 (Goal)
  ↓
工作空間 (Workspace)
  ├─ 任務依賴圖 (Task Graph)
  ├─ 執行實例 (Agent Runs)
  ├─ 作用域審批 (Scoped Approvals)
  └─ 環境沙箱 (Environment Sessions)
  ↓
可驗證交付產物 (Artifacts) 與執行軌跡 (Trajectories)
```

### 2. 編碼執行器治理 (Executor Governance)
像 Claude Code、Codex、OpenCode 等前沿編碼 Agent 非常強大，但在本地「裸跑」 shell 存在巨大的安全隱患，且其執行過程缺少生命週期監管。CoreLayer 提供適配器，將它們收攏為**受管執行器**：
- **邊界防線**：在副作用發生前，強行將改動隔離在 git-worktree 或隔離沙箱會話中。
- **細粒度審批**：分類高風險操作（如刪除、寫入檔案、外發請求），由權限守護進程（Permission Guard）進行非同步掛起並請求用戶手動批准。
- **產物與日誌分離**：最終的可交付代碼/報告（Artifact）被清洗提取，與執行過程中的編譯噪音、狀態日誌徹底分離。

### 3. 「Jarvis 管理工作，專用工具執行工作」
這是 CoreLayer 的核心設計哲學。我們不重複製造輪子，而是提供協調層。Jarvis 負責規劃任務與管理安全，而專門的適配器、工具或 MCP 服務則負責具體執行，使得整個系統不被綁定到任何單一的大模型廠商或 IDE 之中。

---

## 核心架構要素 (Core Pillars)

- **MCP 優先整合** — 原生支援 stdio、HTTP 與 SSE 協定，直接連接個人的本地應用程式和外部服務，並將工具標準化為 `mcp:{serverId}:{toolName}` 格式。
- **權限優先執行階段** — 按低、中、高、極高四大風險級別分類所有工具呼叫，提供決策記憶（Permission Memories）以防頻繁彈窗，並保留完整的稽核日誌。
- **統一工具註冊中心 (Tool Registry)** — 將原生 Rust 模組、MCP 工具、本地技能（Skills）和 REST 適配器統一收攏。
- **模型閘道與智能路由** — 跨 MiMo、Groq、DeepSeek、OpenAI、Gemini、Qwen 以及本地 Ollama 路由請求，為低延遲、工具呼叫或長推理匹配最合適的端點。
- **語音原生管線** — 整合喚醒詞檢測、ASR 語音轉文字、串流 TTS 句級播報，並支援中途打斷插話（Barge-in）。
- **本地優先三模儲存** — 預設採用零配置的本地 SQLite 資料庫，同時在控制中心支援雲端 Supabase 即時同步以及自託管 PostgreSQL 之間的熱切換。
