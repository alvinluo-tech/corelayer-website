---

CoreLayer 在元件之間實施嚴格的邊界，以確保安全、可靠和可維護性。

## 邊界模型

```text
前端（瀏覽器/Webview）
  ║ Tauri IPC（型別化、序列化）
  ╚═► Rust 後端
      ║ HTTP（僅限本機）
      ╚═► Node.js 守護程式
          ║ SDK 介面
          ╚═► 共享套件
```

每個邊界實施：

- **型別安全** — 層之間的型別化介面
- **隔離** — 元件不能直接存取彼此的內部
- **通訊協定** — 僅限定義明確的 API

## 前端 ↔ Rust

通訊使用 Tauri 的 IPC 機制：

- 指令是型別化的（TypeScript ↔ Rust）
- 序列化為 JSON
- 預設為非同步

## Rust ↔ 守護程式

通訊使用 HTTP：

- 僅限本機（不暴露到網路）
- RESTful 端點
- 串流回應（SSE）用於聊天

## 守護程式 ↔ 套件

通訊使用 TypeScript 匯入：

- 直接函式呼叫
- 共享型別定義
- 模組系統實施的套件邊界

## 守護程式 ↔ 外部服務

- **AI 供應商** — HTTPS 到雲端 API
- **MCP 伺服器** — stdio 或 SSE 協定
- **資料庫** — 原生驅動程式（better-sqlite3、pg）

## 為什麼邊界很重要

- **安全** — 被入侵的前端不能直接存取資料庫
- **可靠性** — 守護程式崩潰不會拖垮前端
- **可測試性** — 每一層都可以獨立測試
- **可替換性** — 更換儲存或模型供應商而不需要更改其他層
