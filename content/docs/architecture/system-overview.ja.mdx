CoreLayerは、ユーザーインターフェース、ビジネスロジック、データアクセスを分離する三層プロキシモデルを使用します。

## アーキテクチャ図

```text
┌─────────────────────────────────────────────┐
│              Tauriデスクトップアプリ              │
│  ┌───────────────────────────────────────┐  │
│  │     React 19 + Vite + Tailwind 4     │  │
│  │         （フロントエンドUIレイヤー）           │  │
│  └──────────────┬────────────────────────┘  │
│                 │ Tauri IPC                  │
│  ┌──────────────┴────────────────────────┐  │
│  │      Rustバックエンド（lib.rs）            │  │
│  │   デーモンスーパーバイザー＋HTTPプロキシ      │  │
│  └──────────────┬────────────────────────┘  │
└─────────────────┼───────────────────────────┘
                  │ HTTP
┌─────────────────┴───────────────────────────┐
│          Node.jsデーモン（Hono）              │
│  ┌──────────┐ ┌──────────┐ ┌─────────────┐ │
│  │  Model   │ │   Tool   │ │ Permission  │ │
│  │ Gateway  │ │ Registry │ │   Guard     │ │
│  └──────────┘ └──────────┘ └─────────────┘ │
│  ┌──────────┐ ┌──────────┐ ┌─────────────┐ │
│  │   MCP    │ │ Runtime  │ │  Storage    │ │
│  │  Client  │ │  Core    │ │   Layer     │ │
│  └──────────┘ └──────────┘ └─────────────┘ │
└─────────────────────────────────────────────┘
         │              │              │
    MCPサーバー    AIプロバイダー    データベース
```

## レイヤーの責任

### フロントエンド（React）

- ユーザーインターフェースのレンダリング
- 状態管理（Zustand）
- ユーザー入力処理
- AIレスポンスとツール結果の表示

### Tauriブリッジ（Rust）

- フロントエンドとデーモン間のIPC
- デーモンライフサイクル管理（開始、停止、再起動）
- ネイティブシステム統合（ファイルシステム、通知）
- デーモンへのHTTPプロキシ

### デーモン（Node.js）

- AIオーケストレーションとモデルルーティング
- ツール登録と実行
- 権限チェックと監査ログ
- MCPサーバー管理
- データ永続化

## 通信フロー

```text
ユーザーがメッセージを入力
  → Reactフロントエンドが入力をキャプチャ
  → Tauri IPCがRustバックエンドに送信
  → RustがデーモンにHTTPプロキシ
  → デーモンがJarvisで処理
  → レスポンスが同じパスを通じてストリーミングで返却
  → フロントエンドがレスポンスをレンダリング
```

## パッケージアーキテクチャ

デーモンはモノレポの共有パッケージを利用します:

| パッケージ | 役割 |
|---------|------|
| `@jarvis/types` | 共有TypeScript型 |
| `@jarvis/model-gateway` | マルチプロバイダーモデルルーティング |
| `@jarvis/mcp-client` | MCPサーバー接続 |
| `@jarvis/tool-registry` | ツール管理 |
| `@jarvis/permission-guard` | 安全システム |
| `@jarvis/runtime-core` | ランタイム管理 |
| `@jarvis/runtime-protocol` | プロトコル定義 |
| `@jarvis/execution-environment` | 実行抽象化 |
