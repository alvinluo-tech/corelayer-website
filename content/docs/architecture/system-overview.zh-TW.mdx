CoreLayer 使用三層代理模型，將使用者介面、業務邏輯和資料存取分開。

## 架構圖

```text
┌─────────────────────────────────────────────┐
│              Tauri 桌面應用程式              │
│  ┌───────────────────────────────────────┐  │
│  │     React 19 + Vite + Tailwind 4     │  │
│  │         （前端 UI 層）                 │  │
│  └──────────────┬────────────────────────┘  │
│                 │ Tauri IPC                  │
│  ┌──────────────┴────────────────────────┐  │
│  │      Rust 後端（lib.rs）              │  │
│  │   守護程式監督器 + HTTP 代理           │  │
│  └──────────────┬────────────────────────┘  │
└─────────────────┼───────────────────────────┘
                  │ HTTP
┌─────────────────┴───────────────────────────┐
│          Node.js 守護程式（Hono）            │
│  ┌──────────┐ ┌──────────┐ ┌─────────────┐ │
│  │  模型    │ │   工具   │ │   權限      │ │
│  │  閘道    │ │  註冊表  │ │   守護      │ │
│  └──────────┘ └──────────┘ └─────────────┘ │
│  ┌──────────┐ ┌──────────┐ ┌─────────────┐ │
│  │   MCP    │ │ 執行階段 │ │   儲存      │ │
│  │  用戶端  │ │   核心   │ │    層       │ │
│  └──────────┘ └──────────┘ └─────────────┘ │
└─────────────────────────────────────────────┘
         │              │              │
    MCP 伺服器     AI 供應商       資料庫
```

## 層級職責

### 前端（React）

- 使用者介面渲染
- 狀態管理（Zustand）
- 使用者輸入處理
- 顯示 AI 回應和工具結果

### Tauri 橋接（Rust）

- 前端和守護程式之間的 IPC
- 守護程式生命週期管理（啟動、停止、重新啟動）
- 原生系統整合（檔案系統、通知）
- 到守護程式的 HTTP 代理

### 守護程式（Node.js）

- AI 編排和模型路由
- 工具註冊和執行
- 權限檢查和稽核日誌
- MCP 伺服器管理
- 資料持久化

## 通訊流程

```text
使用者輸入訊息
  → React 前端擷取輸入
  → Tauri IPC 傳送到 Rust 後端
  → Rust 代理 HTTP 到守護程式
  → 守護程式透過 Jarvis 處理
  → 回應透過相同路徑串流回傳
  → 前端渲染回應
```

## 套件架構

守護程式使用來自 monorepo 的共享套件：

| 套件 | 角色 |
|---------|------|
| `@jarvis/types` | 共享 TypeScript 型別 |
| `@jarvis/model-gateway` | 多供應商模型路由 |
| `@jarvis/mcp-client` | MCP 伺服器連接 |
| `@jarvis/tool-registry` | 工具管理 |
| `@jarvis/permission-guard` | 安全系統 |
| `@jarvis/runtime-core` | 執行階段管理 |
| `@jarvis/runtime-protocol` | 協定定義 |
| `@jarvis/execution-environment` | 執行抽象層 |
