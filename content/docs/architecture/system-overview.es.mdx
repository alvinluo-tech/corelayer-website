## Responsabilidades de Capa

### Frontend (React)

- Renderizado de la interfaz de usuario
- Gestion de estado (Zustand)
- Manejo de entrada del usuario
- Presentacion de respuestas de IA y resultados de herramientas

### Puente Tauri (Rust)

- IPC entre frontend y daemon
- Gestion del ciclo de vida del daemon (inicio, parada, reinicio)
- Integracion nativa del sistema (sistema de archivos, notificaciones)
- Proxy HTTP al daemon

### Daemon (Node.js)

- Orquestacion de IA y enrutamiento de modelos
- Registro y ejecucion de herramientas
- Verificacion de permisos y registro de auditoria
- Gestion de servidores MCP
- Persistencia de datos

## Flujo de Comunicacion

```text
El usuario escribe un mensaje
  вҶ’ El frontend React captura la entrada
  вҶ’ Tauri IPC envia al backend Rust
  вҶ’ Rust hace proxy HTTP al daemon
  вҶ’ El daemon procesa con Jarvis
  вҶ’ La respuesta se transmite de vuelta por la misma ruta
  вҶ’ El frontend renderiza la respuesta
```

## Arquitectura de Paquetes

El daemon consume paquetes compartidos del monorepo:

| Paquete | Rol |
|---------|------|
| `@jarvis/types` | Tipos TypeScript compartidos |
| `@jarvis/model-gateway` | Enrutamiento multi-proveedor de modelos |
| `@jarvis/mcp-client` | Conexiones a servidores MCP |
| `@jarvis/tool-registry` | Gestion de herramientas |
| `@jarvis/permission-guard` | Sistema de seguridad |
| `@jarvis/runtime-core` | Gestion de runtime |
| `@jarvis/runtime-protocol` | Definiciones de protocolo |
| `@jarvis/execution-environment` | Abstraccion de ejecucion |
