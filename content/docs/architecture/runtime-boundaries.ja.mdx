title: ランタイム境界
description: CoreLayerが安全性と信頼性のためにコンポーネントを分離する方法。
---

CoreLayerは、安全性、信頼性、保守性を確保するために、コンポーネント間に厳格な境界を適用します。

## 境界モデル

```text
フロントエンド（ブラウザ/Webview）
  ║ Tauri IPC（型付き、シリアライズ済み）
  ╚═► Rustバックエンド
      ║ HTTP（ローカルホストのみ）
      ╚═► Node.jsデーモン
          ║ SDKインターフェース
          ╚═► 共有パッケージ
```

各境界が適用するもの:

- **型安全性** — レイヤー間の型付きインターフェース
- **分離** — コンポーネントは互いの内部に直接アクセスできない
- **通信プロトコル** — 定義済みAPIのみ

## フロントエンド ↔ Rust

通信にはTauriのIPCメカニズムを使用:

- コマンドは型付き（TypeScript ↔ Rust）
- JSONにシリアライズ
- デフォルトで非同期

## Rust ↔ デーモン

通信にはHTTPを使用:

- ローカルホストのみ（ネットワークには公開されない）
- RESTfulエンドポイント
- チャット用のストリーミングレスポンス（SSE）

## デーモン ↔ パッケージ

通信にはTypeScriptインポートを使用:

- 直接関数呼び出し
- 共有型定義
- モジュールシステムによるパッケージ境界の適用

## デーモン ↔ 外部サービス

- **AIプロバイダー** — クラウドAPIへのHTTPS
- **MCPサーバー** — stdioまたはSSEプロトコル
- **データベース** — ネイティブドライバー（better-sqlite3、pg）

## 境界が重要な理由

- **セキュリティ** — 侵害されたフロントエンドがデータベースに直接アクセスできない
- **信頼性** — デーモンのクラッシュがフロントエンドをダウンさせない
- **テスト容易性** — 各レイヤーを独立してテスト可能
- **置換可能性** — 他のレイヤーを変更せずにストレージやモデルプロバイダーを交換可能
