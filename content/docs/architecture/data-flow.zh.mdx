本页追踪 CoreLayer 中常见操作的完整数据流。

## 聊天消息流

```text
1. 用户在聊天面板输入消息
2. 前端通过 Tauri IPC 发送
3. Rust 后端代理到守护进程 HTTP API
4. 守护进程在 /api/chat 接收
5. Jarvis 组装上下文：
   - 对话历史
   - 用户记忆
   - 可用工具
6. 模型网关选择提供商
7. 请求发送到 AI 提供商（流式）
8. 响应流式返回：
   - 文本块 → 前端
   - 工具调用 → 权限守卫
9. 如果是工具调用：
   a. 权限守卫检查风险
   b. 如果需要审批 → 暂停，显示面板
   c. 用户批准 → 执行工具
   d. 结果返回给模型
   e. 模型继续响应
10. 最终响应渲染到聊天中
```

## 工具执行流

```text
工具调用请求
  → 工具注册解析工具
  → 权限守卫分类风险
  → 策略检查：
    - allow → 立即执行
    - confirm → 暂停等待审批
    - deny → 阻止并记录
  → 执行工具（MCP/原生/REST）
  → 返回结果给模型
  → 记录到审计日志
```

## MCP 连接流

```text
1. 守护进程读取 MCP 服务器配置
2. 启动子进程（stdio）或通过 SSE 连接
3. MCP 握手（initialize）
4. 服务器报告可用工具
5. 工具注册注册工具
6. 工具对 Jarvis 可用
7. 断开连接时 → 尝试重连
```

## 数据存储流

```text
写操作：
  → 存储层接收写请求
  → 验证数据模式
  → 写入活跃后端（SQLite/Supabase/PostgreSQL）
  → 返回确认

读操作：
  → 存储层接收读请求
  → 查询活跃后端
  → 返回结果
```

## 语音管线流

```text
检测到唤醒词（Picovoice）
  → 开始监听
  → 音频流发送到 ASR 提供商
  → 返回转录文本
  → 文本发送到聊天管线（与输入文字相同）
  → 生成响应
  → TTS 流式返回音频
  → 音频通过系统输出播放
  → 打断：如果用户在播放期间说话 → 从 ASR 重新开始
```
