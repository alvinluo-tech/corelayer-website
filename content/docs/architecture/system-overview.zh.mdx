CoreLayer 使用三层代理模型，将用户界面、业务逻辑和数据访问分离。

## 架构图

```text
┌─────────────────────────────────────────────┐
│              Tauri 桌面应用                  │
│  ┌───────────────────────────────────────┐  │
│  │     React 19 + Vite + Tailwind 4     │  │
│  │           （前端 UI 层）               │  │
│  └──────────────┬────────────────────────┘  │
│                 │ Tauri IPC                  │
│  ┌──────────────┴────────────────────────┐  │
│  │      Rust 后端（lib.rs）               │  │
│  │   守护进程管理器 + HTTP 代理            │  │
│  └──────────────┬────────────────────────┘  │
└─────────────────┼───────────────────────────┘
                  │ HTTP
┌─────────────────┴───────────────────────────┐
│          Node.js 守护进程（Hono）            │
│  ┌──────────┐ ┌──────────┐ ┌─────────────┐ │
│  │  模型    │ │   工具    │ │   权限      │ │
│  │  网关    │ │   注册    │ │   守卫      │ │
│  └──────────┘ └──────────┘ └─────────────┘ │
│  ┌──────────┐ ┌──────────┐ ┌─────────────┐ │
│  │   MCP    │ │  运行时   │ │   存储      │ │
│  │   客户端 │ │   核心    │ │   层        │ │
│  └──────────┘ └──────────┘ └─────────────┘ │
└─────────────────────────────────────────────┘
         │              │              │
    MCP 服务器     AI 提供商       数据库
```

## 各层职责

### 前端（React）

- 用户界面渲染
- 状态管理（Zustand）
- 用户输入处理
- 显示 AI 响应和工具结果

### Tauri 桥接层（Rust）

- 前端与守护进程之间的 IPC
- 守护进程生命周期管理（启动、停止、重启）
- 原生系统集成（文件系统、通知）
- 到守护进程的 HTTP 代理

### 守护进程（Node.js）

- AI 编排和模型路由
- 工具注册和执行
- 权限检查和审计日志
- MCP 服务器管理
- 数据持久化

## 通信流程

```text
用户输入消息
  → React 前端捕获输入
  → Tauri IPC 发送到 Rust 后端
  → Rust 通过 HTTP 代理到守护进程
  → 守护进程通过 Jarvis 处理
  → 响应通过相同路径流式返回
  → 前端渲染响应
```

## 包架构

守护进程使用来自 monorepo 的共享包：

| 包 | 角色 |
|----|------|
| `@jarvis/types` | 共享 TypeScript 类型 |
| `@jarvis/model-gateway` | 多提供商模型路由 |
| `@jarvis/mcp-client` | MCP 服务器连接 |
| `@jarvis/tool-registry` | 工具管理 |
| `@jarvis/permission-guard` | 安全系统 |
| `@jarvis/runtime-core` | 运行时管理 |
| `@jarvis/runtime-protocol` | 协议定义 |
| `@jarvis/execution-environment` | 执行抽象 |
