CoreLayer 在组件之间实施严格的边界，以确保安全性、可靠性和可维护性。

## 边界模型

```text
前端（浏览器/Webview）
  ║ Tauri IPC（类型化、序列化）
  ╚═► Rust 后端
      ║ HTTP（仅限本地主机）
      ╚═► Node.js 守护进程
          ║ SDK 接口
          ╚═► 共享包
```

每个边界强制执行：

- **类型安全** — 层与层之间使用类型化接口
- **隔离** — 组件不能直接访问彼此的内部实现
- **通信协议** — 仅通过定义良好的 API

## 前端 ↔ Rust

通信使用 Tauri 的 IPC 机制：

- 命令是类型化的（TypeScript ↔ Rust）
- 序列化为 JSON
- 默认异步

## Rust ↔ 守护进程

通信使用 HTTP：

- 仅限本地主机（不暴露到网络）
- RESTful 端点
- 流式响应（SSE）用于聊天

## 守护进程 ↔ 包

通信使用 TypeScript 导入：

- 直接函数调用
- 共享类型定义
- 包边界由模块系统强制执行

## 守护进程 ↔ 外部服务

- **AI 提供商** — 通过 HTTPS 连接云端 API
- **MCP 服务器** — stdio 或 SSE 协议
- **数据库** — 原生驱动（better-sqlite3、pg）

## 为什么边界很重要

- **安全** — 被入侵的前端不能直接访问数据库
- **可靠性** — 守护进程崩溃不会导致前端瘫痪
- **可测试性** — 每一层都可以独立测试
- **可替换性** — 可以在不更改其他层的情况下更换存储或模型提供商
