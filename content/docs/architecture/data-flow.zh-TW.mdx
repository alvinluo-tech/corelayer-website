本頁追蹤 CoreLayer 中常見操作的完整資料流程。

## 聊天訊息流程

```text
1. 使用者在聊天面板輸入訊息
2. 前端透過 Tauri IPC 傳送
3. Rust 後端代理到守護程式 HTTP API
4. 守護程式在 /api/chat 接收
5. Jarvis 組裝上下文：
   - 對話歷史
   - 使用者記憶
   - 可用工具
6. 模型閘道選擇供應商
7. 請求傳送到 AI 供應商（串流）
8. 回應串流回傳：
   - 文字區塊 → 前端
   - 工具呼叫 → 權限守護
9. 如果是工具呼叫：
   a. 權限守護檢查風險
   b. 如需核准 → 暫停，顯示面板
   c. 使用者核准 → 執行工具
   d. 結果傳回模型
   e. 模型繼續回應
10. 最終回應在聊天中渲染
```

## 工具執行流程

```text
工具呼叫請求
  → 工具註冊表解析工具
  → 權限守護分類風險
  → 策略檢查：
    - 允許 → 立即執行
    - 確認 → 暫停等待核准
    - 拒絕 → 封鎖並記錄
  → 執行工具（MCP/原生/REST）
  → 將結果傳回模型
  → 記錄到稽核軌跡
```

## MCP 連接流程

```text
1. 守護程式讀取 MCP 伺服器設定
2. 產生子程序（stdio）或透過 SSE 連接
3. MCP 握手（初始化）
4. 伺服器報告可用工具
5. 工具註冊表註冊工具
6. 工具可供 Jarvis 使用
7. 斷線時 → 嘗試重新連接
```

## 資料儲存流程

```text
寫入操作：
  → 儲存層接收寫入請求
  → 驗證資料結構
  → 寫入活動後端（SQLite/Supabase/PostgreSQL）
  → 回傳確認

讀取操作：
  → 儲存層接收讀取請求
  → 查詢活動後端
  → 回傳結果
```

## 語音管線流程

```text
偵測到喚醒詞（Picovoice）
  → 開始聆聽
  → 音訊串流到 ASR 供應商
  → 回傳轉錄文字
  → 文字傳送到聊天管線（與輸入文字相同）
  → 產生回應
  → TTS 串流音訊回傳
  → 音訊透過系統輸出播放
  → 打斷：如果使用者在播放期間說話 → 從 ASR 重新開始
```
