このページでは、CoreLayerの一般的な操作における完全なデータフローを追跡します。

## チャットメッセージフロー

```text
1. ユーザーがチャットパネルにメッセージを入力
2. フロントエンドがTauri IPC経由で送信
3. RustバックエンドがデーモンHTTP APIにプロキシ
4. デーモンが/api/chatで受信
5. Jarvisがコンテキストを構築:
   - 会話履歴
   - ユーザーメモリ
   - 利用可能なツール
6. モデルゲートウェイがプロバイダーを選択
7. AIプロバイダーにリクエスト送信（ストリーミング）
8. レスポンスがストリーミングで返却:
   - テキストチャンク → フロントエンド
   - ツール呼び出し → パーミッションガード
9. ツール呼び出しの場合:
   a. パーミッションガードがリスクをチェック
   b. 承認が必要な場合 → 一時停止、パネル表示
   c. ユーザーが承認 → ツール実行
   d. 結果をモデルに返送
   e. モデルがレスポンスを継続
10. 最終レスポンスをチャットにレンダリング
```

## ツール実行フロー

```text
ツール呼び出し要求
  → ツールレジストリがツールを解決
  → パーミッションガードがリスクを分類
  → ポリシーチェック:
    - allow → 即座に実行
    - confirm → 承認待ちで一時停止
    - deny → ブロックしてログ記録
  → ツール実行（MCP/ネイティブ/REST）
  → 結果をモデルに返却
  → 監査証跡にログ記録
```

## MCP接続フロー

```text
1. デーモンがMCPサーバー設定を読み込む
2. 子プロセス生成（stdio）またはSSE経由で接続
3. MCPハンドシェイク（initialize）
4. サーバーが利用可能なツールを報告
5. ツールレジストリがツールを登録
6. Jarvisがツールを利用可能に
7. 切断時 → 再接続を試行
```

## データストレージフロー

```text
書き込み操作:
  → ストレージレイヤーが書き込みリクエストを受信
  → データスキーマを検証
  → アクティブバックエンドに書き込み（SQLite/Supabase/PostgreSQL）
  → 確認を返却

読み取り操作:
  → ストレージレイヤーが読み取りリクエストを受信
  → アクティブバックエンドにクエリ
  → 結果を返却
```

## 音声パイプラインフロー

```text
ウェイクワード検出（Picovoice）
  → リスニング開始
  → ASRプロバイダーへのオーディオストリーム
  → 文字起こしが返却
  → テキストをチャットパイプラインに送信（タイプ入力と同じ）
  → レスポンス生成
  → TTSがオーディオをストリーミングで返却
  → システム出力からオーディオ再生
  → バージイン: 再生中にユーザーが話した場合 → ASRから再開
```
