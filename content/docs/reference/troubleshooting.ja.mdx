## アプリが起動しない

**症状:** CoreLayerが起動しない、またはすぐにクラッシュする。

**解決策:**
- Node.js 22以上がインストールされているか確認: `node --version`
- pnpm 9以上がインストールされているか確認: `pnpm --version`
- プラットフォーム固有のTauri前提条件を確認
- リビルドを試す: `pnpm clean && pnpm build`

## デーモンが起動しない

**症状:** "Daemon unhealthy"または接続エラー。

**解決策:**
- ポート3001が使用されていないか確認: macOS/Linuxでは`lsof -i :3001`、Windowsでは`netstat -ano | findstr 3001`
- 競合プロセスを終了: `pnpm clean:daemon`
- コントロールセンターでデーモンログを確認
- データベースファイルの権限を確認

## モデルが応答しない

**症状:** Jarvisが応答しない、または"No model available"と表示される。

**解決策:**
- 設定 → モデルでAPIキーが正しいか確認
- プロバイダーのステータスを確認（Groq、OpenRouterなど）
- 別のモデル/プロバイダーを試す
- ネットワーク接続を確認

## MCPサーバーが接続されない

**症状:** サーバーが"Disconnected"または"Error"ステータスを表示。

**解決策:**
- コマンドとargsが正しいか確認
- ターミナルでコマンドを手動テスト
- npxがパッケージをインストールできるか確認: `npx -y @modelcontextprotocol/server-filesystem`
- 環境変数が設定されているか確認
- コントロールセンターからMCP接続を再起動

## 音声が動作しない

**症状:** ウェイクワードが検出されない、または音声が認識されない。

**解決策:**
- システム設定でマイクの権限を確認
- バックグラウンドノイズを軽減
- ウェイクワードの代わりにマイクボタンをクリックして試す
- より高い精度のためにASRプロバイダーをGroq Whisperに切り替え
- TTSのオーディオ出力設定を確認

## パーミッションガードがすべてをブロックする

**症状:** すべてのツール呼び出しに承認が必要。

**解決策:**
- 設定 → 権限で権限ポリシーを確認
- 信頼するツールに`allow`ルールを追加
- 信頼するツールに「常に承認」を使用

## ビルド失敗

**症状:** `pnpm build`が失敗する。

**解決策:**
- クリーンしてリビルド: `pnpm clean && pnpm install && pnpm build`
- Rustツールチェーンを確認: `rustc --version`、`cargo --version`
- Tauriのプラットフォーム前提条件を確認
- TypeScriptエラーを確認: `pnpm typecheck`

## データベースの問題

**症状:** データが保存されない、または読み込みエラー。

**解決策:**
- 設定 → ストレージでストレージ設定を確認
- データベースファイルが存在し書き込み可能か確認
- 別のストレージモードに切り替えてみる
- Supabaseの場合: URLとanonキーを確認

## まだ解決しない場合？

- 既知の問題については[GitHub Issues](https://github.com/alvinluo-tech/CoreLayer/issues)を確認
- 新しいissueを作成:
  - お使いのプラットフォームとバージョン
  - 再現手順
  - エラーメッセージまたはログ
  - 期待される動作と実際の動作
