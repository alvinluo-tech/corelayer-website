## 應用程式無法啟動

**症狀：** CoreLayer 啟動失敗或立即崩潰。

**解決方案：**
- 驗證已安裝 Node.js 22+：`node --version`
- 驗證已安裝 pnpm 9+：`pnpm --version`
- 檢查平台特定的 Tauri 前置條件
- 嘗試重新建置：`pnpm clean && pnpm build`

## 守護程式無法啟動

**症狀：**「守護程式不健康」或連接錯誤。

**解決方案：**
- 檢查連接埠 3001 是否被佔用：`lsof -i :3001`（macOS/Linux）或 `netstat -ano | findstr 3001`（Windows）
- 終止衝突的程序：`pnpm clean:daemon`
- 檢查控制中心中的守護程式日誌
- 驗證資料庫檔案權限

## 模型無回應

**症狀：** Jarvis 沒有回應或顯示「無可用模型」。

**解決方案：**
- 在設定 → 模型中驗證 API 金鑰是否正確
- 檢查供應商狀態（Groq、OpenRouter 等）
- 嘗試不同的模型/供應商
- 檢查網路連接

## MCP 伺服器無法連接

**症狀：** 伺服器顯示「已斷線」或「錯誤」狀態。

**解決方案：**
- 驗證指令和參數是否正確
- 在終端機中手動測試指令
- 檢查 npx 是否可以安裝套件：`npx -y @modelcontextprotocol/server-filesystem`
- 檢查環境變數是否已設定
- 從控制中心重新啟動 MCP 連接

## 語音無法運作

**症狀：** 喚醒詞未偵測或語音未辨識。

**解決方案：**
- 在系統設定中檢查麥克風權限
- 減少背景噪音
- 嘗試按一下麥克風按鈕而不是使用喚醒詞
- 將 ASR 供應商切換到 Groq Whisper 以獲得更好的準確度
- 檢查 TTS 的音訊輸出設定

## 權限守護封鎖所有東西

**症狀：** 所有工具呼叫都需要核准。

**解決方案：**
- 在設定 → 權限中檢查權限策略
- 為受信任的工具新增 `allow` 規則
- 對你信任的工具使用「一律核准」

## 建置失敗

**症狀：** `pnpm build` 失敗。

**解決方案：**
- 清理並重新建置：`pnpm clean && pnpm install && pnpm build`
- 檢查 Rust 工具鏈：`rustc --version`、`cargo --version`
- 驗證 Tauri 的平台前置條件
- 檢查 TypeScript 錯誤：`pnpm typecheck`

## 資料庫問題

**症狀：** 資料未儲存或載入錯誤。

**解決方案：**
- 在設定 → 儲存中檢查儲存設定
- 驗證資料庫檔案存在且可寫入
- 嘗試切換到不同的儲存模式
- 對於 Supabase：驗證 URL 和匿名金鑰

## 仍然無法解決？

- 檢查 [GitHub Issues](https://github.com/alvinluo-tech/CoreLayer/issues) 了解已知問題
- 開啟新 issue，包含：
  - 你的平台和版本
  - 重現步驟
  - 錯誤訊息或日誌
  - 預期行為與實際行為
