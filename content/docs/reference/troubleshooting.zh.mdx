## 应用无法启动

**症状：** CoreLayer 启动失败或立即崩溃。

**解决方案：**
- 验证 Node.js 22+ 已安装：`node --version`
- 验证 pnpm 9+ 已安装：`pnpm --version`
- 检查平台特定的 Tauri 前提条件
- 尝试重新构建：`pnpm clean && pnpm build`

## 守护进程未启动

**症状：** "守护进程不健康"或连接错误。

**解决方案：**
- 检查端口 3001 是否被占用：`lsof -i :3001`（macOS/Linux）或 `netstat -ano | findstr 3001`（Windows）
- 终止冲突进程：`pnpm clean:daemon`
- 在控制中心检查守护进程日志
- 验证数据库文件权限

## 模型无响应

**症状：** Jarvis 不响应或显示"无可用模型"。

**解决方案：**
- 在设置 → 模型中验证 API 密钥是否正确
- 检查提供商状态（Groq、OpenRouter 等）
- 尝试不同的模型/提供商
- 检查网络连接

## MCP 服务器无法连接

**症状：** 服务器显示"已断开"或"错误"状态。

**解决方案：**
- 验证命令和参数是否正确
- 在终端中手动测试命令
- 检查 npx 是否可以安装包：`npx -y @modelcontextprotocol/server-filesystem`
- 检查环境变量是否已设置
- 从控制中心重启 MCP 连接

## 语音不工作

**症状：** 唤醒词未检测到或语音未识别。

**解决方案：**
- 在系统设置中检查麦克风权限
- 减少背景噪音
- 尝试点击麦克风按钮而不是使用唤醒词
- 切换 ASR 提供商到 Groq Whisper 以获得更好的准确度
- 检查 TTS 的音频输出设置

## 权限守卫阻止一切

**症状：** 所有工具调用都需要审批。

**解决方案：**
- 在设置 → 权限中检查权限策略
- 为受信任的工具添加 `allow` 规则
- 对你信任的工具使用"始终批准"

## 构建失败

**症状：** `pnpm build` 失败。

**解决方案：**
- 清理并重新构建：`pnpm clean && pnpm install && pnpm build`
- 检查 Rust 工具链：`rustc --version`、`cargo --version`
- 验证 Tauri 的平台前提条件
- 检查 TypeScript 错误：`pnpm typecheck`

## 数据库问题

**症状：** 数据未保存或加载错误。

**解决方案：**
- 在设置 → 存储中检查存储配置
- 验证数据库文件存在且可写
- 尝试切换到不同的存储模式
- 对于 Supabase：验证 URL 和匿名密钥
