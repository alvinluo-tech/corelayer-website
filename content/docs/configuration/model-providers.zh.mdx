---
title: 模型提供商
description: 配置 Jarvis 路由的 AI 模型提供商。
---

CoreLayer 的模型网关跨多个 AI 提供商路由请求。你可以配置一个或多个提供商，让 Jarvis 为每个任务选择最佳模型。

## 支持的提供商

模型网关通过内建预设原生支持以下大模型提供商：

| 提供商 | ID | 类型 | 常用模型 |
|--------|----|------|----------|
| **MiMo (小米)** | `mimo` | `openai_compatible` | `mimo-v2.5-pro`, `mimo-2.5` |
| **Groq** | `groq` | `openai_compatible` | `llama-3.3-70b-versatile`, `mixtral-8x7b-32768` |
| **DeepSeek** | `deepseek` | `openai_compatible` | `deepseek-chat`, `deepseek-reasoner` |
| **Anthropic** | `anthropic` | `openai_compatible` | `claude-sonnet-4`, `claude-haiku-4-5` |
| **Google Gemini** | `google` | `openai_compatible` | `gemini-2.5-pro`, `gemini-2.5-flash` |
| **OpenAI** | `openai` | `openai_compatible` | `gpt-4o`, `gpt-4o-mini`, `o3-mini` |
| **通义千问 (Qwen)** | `qwen` | `openai_compatible` | `qwen-max`, `qwen-plus` |
| **OpenRouter** | `openrouter` | `openai_compatible` | `anthropic/claude-sonnet-4`, `google/gemini-2.5-pro` |
| **Ollama** | `ollama` | `ollama` | `llama3.2`, `qwen2.5`, `deepseek-r1` |

## 配置文件

CoreLayer 采用**配置与密钥分离**的安全设计。配置文件保存在 `~/.jarvis/config/` 目录下。

### 1. 基础配置 (`config.json`)

定义激活的提供商、默认模型和各个提供商的端点，此文件**不包含任何 API 密钥**：

```json
{
  "version": 1,
  "activeProvider": "mimo",
  "activeModel": "mimo-v2.5-pro",
  "providers": [
    {
      "id": "mimo",
      "name": "MiMo",
      "type": "openai_compatible",
      "baseURL": "https://token-plan-ams.xiaomimimo.com/v1",
      "enabled": true
    },
    {
      "id": "groq",
      "name": "Groq",
      "type": "openai_compatible",
      "baseURL": "https://api.groq.com/openai/v1",
      "enabled": true
    },
    {
      "id": "openrouter",
      "name": "OpenRouter",
      "type": "openai_compatible",
      "baseURL": "https://openrouter.ai/api/v1",
      "enabled": true
    },
    {
      "id": "ollama",
      "name": "Ollama",
      "type": "ollama",
      "baseURL": "http://localhost:11434/v1",
      "enabled": true
    }
  ]
}
```

### 2. 密钥配置 (`credentials.json`)

独立保存密钥，与 `config.json` 中的 `id` 一一对应。

```json
{
  "mimo": "your_mimo_api_key_here",
  "groq": "your_groq_api_key_here",
  "openrouter": "your_openrouter_api_key_here"
}
```

## 模型路由

当配置了多个提供商时，模型网关根据以下条件选择：

- **任务复杂度** — 复杂推理路由到更强大的模型
- **延迟要求** — 时间敏感的请求优先使用 Groq 或本地模型
- **成本优化** — 常规任务使用更便宜的提供商
- **可用性** — 当提供商宕机时进行回退

## 环境变量

你可以使用环境变量而不是硬编码 API 密钥：

```bash
GROQ_API_KEY=your_groq_api_key_here
OPENROUTER_API_KEY=your_openrouter_api_key_here
```

在配置中引用它们：

```json
{
  "apiKey": "${GROQ_API_KEY}"
}
```
