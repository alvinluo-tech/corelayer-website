CoreLayerには、音声でJarvisとやり取りできる音声パイプラインが含まれています。

## 音声パイプライン概要

```text
ウェイクワード検出
  → 音声認識（ASR）
  → 意図処理（LLM）
  → テキスト読み上げ（TTS）
  → オーディオ出力

  ↕ 任意のタイミングでバージイン割り込み
```

## 設定

```json
{
  "voice": {
    "enabled": true,
    "wakeWord": {
      "enabled": true,
      "keyword": "jarvis"
    },
    "asr": {
      "provider": "web-speech",
      "language": "en-US"
    },
    "tts": {
      "provider": "mimo",
      "streaming": true,
      "sentenceLevel": true
    }
  }
}
```

## ウェイクワード

Picovoice Porcupineを使用したウェイクワード検出:

- ブラウザ/Webview内でローカル実行
- アイドル時のCPU使用率が低い
- 設定可能なキーワード（デフォルト: "jarvis"）

## 音声認識（ASR）

2つのプロバイダーが利用可能:

| プロバイダー | タイプ | 品質 |
|----------|------|---------|
| **Web Speech API** | ブラウザネティブ | 良好、APIキー不要 |
| **Groq Whisper** | クラウド | 高精度、APIキーが必要 |

## テキスト読み上げ（TTS）

| プロバイダー | タイプ | 機能 |
|----------|------|----------|
| **MiMo TTS**（`mimo`） | クラウド | 文レベルストリーミング、自然な音声 |

MiMo TTSはクラウドベースのテキスト読み上げサービスで、文レベルのストリーミングをサポート — レスポンス全体を待つ代わりに、文が生成されるとすぐに読み上げを開始します。これにより知覚レイテンシーが大幅に軽減されます。

ストリーミングTTSはレスポンスが生成されると同時に読み上げを行い、知覚レイテンシーを軽減します。

## バージイン割り込み

有効にすると、Jarvisのレスポンス中に話しかけて割り込めます。パイプラインは以下を行います:

1. 現在のTTS再生を停止
2. 新しい入力を処理
3. 割り込みに応答

## 音声オーバーレイ

音声がアクティブな場合、フローティングオーバーレイが表示されます:

- 現在の状態（リスニング、処理中、スピーキング）
- 波形ビジュアライゼーション
- 文字起こしテキスト
