CoreLayerは、ランタイム中にホットスワップ可能な3つのストレージバックエンドをサポートします。

## ストレージモード

| モード | 説明 | ユースケース |
|------|-------------|----------|
| **SQLite** | ローカルファイルデータベース（デフォルト） | 個人利用、オフラインファースト |
| **Supabase** | リアルタイム同期付きクラウドPostgreSQL | マルチデバイス同期 |
| **PostgreSQL** | セルフホストPostgreSQL | チーム/エンタープライズ向け |

## SQLite（デフォルト）

SQLiteはデフォルトのストレージモードです。データはローカルの`.db`ファイルに保存されます:

```json
{
  "storage": {
    "mode": "sqlite",
    "path": "~/.corelayer/data.db"
  }
}
```

外部依存は不要です。完全にオフラインで動作します。

## Supabase

クラウド同期のためにSupabaseに接続:

```json
{
  "storage": {
    "mode": "supabase",
    "url": "https://your-project.supabase.co",
    "anonKey": "eyJ..."
  }
}
```

Supabaseのセットアップ:
1. [supabase.com](https://supabase.com)でプロジェクトを作成
2. CoreLayerマイグレーションスクリプトを実行
3. プロジェクトURLとanonキーを追加

## PostgreSQL

セルフホストのPostgreSQLインスタンスに接続:

```json
{
  "storage": {
    "mode": "postgres",
    "connectionString": "postgresql://user:pass@localhost:5432/corelayer"
  }
}
```

## ホットスイッチング

コントロールセンターからランタイム中にストレージモードを切り替えられます:

1. **設定 → ストレージ**を開く
2. 新しいモードを選択
3. 接続詳細を入力
4. **切り替え**をクリック

CoreLayerがモード間のデータ移行を自動的に行います。

## データスキーマ (Data Schema)

CoreLayerは高度に構造化されたローカルファーストのデータベーススキーマを使用しており、以下の主要エンティティを保存します:

*   **workspaces** — 目標、タスク分解、および実行状態を管理する最上位のワークスペース
*   **projects** — 仕様書（specs）、技術スタック、およびローカルのルートパスを含む、ワークスペース内の特定のプロジェクトフォルダー
*   **tasks** — 実行の依存関係グラフを形成するタスクノード
*   **artifacts** — 価値のあるファイルやレポートをターミナル出力やログのノイズから分離した最終的な成果物
*   **agent_runs** — エージェントの個別実行セッション（状態、遅延、ルーティング、およびログを追跡）
*   **approval_requests** — ユーザーの手動承認の決定を待機する保留中の高リスクアクション
*   **permission_memories** — 繰り返しの権限確認ポップアップを防ぐための決定の一時的なキャッシュ
*   **conversations** / **messages** — チャット会話の履歴と個々のメッセージレコード
*   **tool_call_logs** — 実行時間、引数、リスクレベル、および結果を記録する監査ログ
*   **app_connections** — 設定されたローカルまたはリモートのMCPサーバーおよびアプリケーション接続
*   **model_profiles** — モデルゲートウェイのプロバイダー設定とインターフェースエンドポイント
*   **memories** — Jarvisの長期メモリ（事実、好み、コンテキスト、固定済み）
*   **scheduled_tasks** — 自動化タスクとスキルをトリガーするCronジョブ
*   **environment_sessions** / **environment_events** — 隔離されたサンドボックスコンテナとイベントストリーム
*   **articles** — リーディングリストのエントリ
*   **reviews** — 日次/週次レビューの要約
