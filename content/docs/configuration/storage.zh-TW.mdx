CoreLayer 支援三種儲存後端，可在執行階段進行熱切換。

## 儲存模式

| 模式 | 說明 | 使用情境 |
|------|-------------|----------|
| **SQLite** | 本地檔案資料庫（預設） | 個人使用、離線優先 |
| **Supabase** | 具有即時同步的雲端 PostgreSQL | 多裝置同步 |
| **PostgreSQL** | 自架 PostgreSQL | 團隊/企業使用 |

## SQLite（預設）

SQLite 是預設的儲存模式。資料儲存在本地 `.db` 檔案中：

```json
{
  "storage": {
    "mode": "sqlite",
    "path": "~/.corelayer/data.db"
  }
}
```

無需外部依賴。完全離線運作。

## Supabase

連接到 Supabase 以進行雲端同步：

```json
{
  "storage": {
    "mode": "supabase",
    "url": "https://your-project.supabase.co",
    "anonKey": "eyJ..."
  }
}
```

設定 Supabase：
1. 在 [supabase.com](https://supabase.com) 建立專案
2. 執行 CoreLayer 遷移腳本
3. 新增你的專案 URL 和匿名金鑰

## PostgreSQL

連接到自架的 PostgreSQL 實例：

```json
{
  "storage": {
    "mode": "postgres",
    "connectionString": "postgresql://user:pass@localhost:5432/corelayer"
  }
}
```

## 熱切換

你可以從控制中心在執行階段切換儲存模式：

1. 開啟 **設定 → 儲存**
2. 選擇新模式
3. 輸入連接詳情
4. 按一下 **切換**

CoreLayer 會自動在模式之間遷移資料。

## 資料模式 (Data Schema)

CoreLayer 採用本地優先的高完整度資料庫表設計，儲存以下主要實體：

*   **workspaces** — 頂層工作空間，管理用戶目標、任務拆解與執行階段狀態
*   **projects** — 工作空間內的項目，包含項目規格（spec）、技術棧及本地根路徑
*   **tasks** — 任務節點，形成任務圖（Task Graph）並追蹤 Agent 執行依賴
*   **artifacts** — 交付產物，將最終有價值的交付檔案/報告與執行過程中的日誌及噪音分離
*   **agent_runs** — 單次 Agent 運行記錄，追蹤狀態、耗時、模型路由與事件日誌
*   **approval_requests** — 掛起的高風險操作，記錄用戶對工具執行的安全審查決策
*   **permission_memories** — 權限快取記憶，針對特定項目或會話臨時授權以防頻繁彈窗
*   **conversations** / **messages** — 聊天對話歷史及單條訊息記錄
*   **tool_call_logs** — 稽核日誌，記錄每次工具呼叫的耗時、參數、風險等級及結果
*   **app_connections** — 已經設定的本地/遠端 MCP 伺服器與外部應用程式連接
*   **model_profiles** — 模型閘道的供應商設定與介面端點
*   **memories** — Jarvis 的長期記憶（facts、preferences、context、pinned）
*   **scheduled_tasks** — 定時觸發的自動化 Cron 任務與技能
*   **environment_sessions** / **environment_events** — 隔離的開發沙箱環境會話及事件流
*   **articles** — 閱讀清單條目
*   **reviews** — 每日/每週回顧摘要
