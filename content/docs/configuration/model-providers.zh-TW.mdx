CoreLayer 的模型閘道跨多個 AI 供應商路由請求。你可以設定一個或多個供應商，讓 Jarvis 為每個任務選擇最佳模型。

## 支援的供應商

模型閘道透過內建預設原生支援以下大模型供應商：

| 供應商 | ID | 類型 | 常用模型 |
|--------|----|------|----------|
| **MiMo (小米)** | `mimo` | `openai_compatible` | `mimo-v2.5-pro`, `mimo-v2.5` |
| **Groq** | `groq` | `openai_compatible` | `llama-4-scout`, `llama-4-maverick`, `llama-3.3-70b-versatile` |
| **DeepSeek** | `deepseek` | `openai_compatible` | `deepseek-v4-pro`, `deepseek-v4-flash` |
| **Anthropic** | `anthropic` | `openai_compatible` | `claude-sonnet-5`, `claude-fable-5`, `claude-opus-4.8` |
| **Google Gemini** | `google` | `openai_compatible` | `gemini-3.5-flash`, `gemini-3.1-flash`, `gemini-3.1-pro` |
| **OpenAI** | `openai` | `openai_compatible` | `gpt-5.6-sol`, `gpt-5.6-terra`, `gpt-5.5`, `gpt-5.4` |
| **通義千問 (Qwen)** | `qwen` | `openai_compatible` | `qwen-3.7-max-preview`, `qwen-3.7-plus-preview`, `qwen-3.6` |
| **OpenRouter** | `openrouter` | `openai_compatible` | `anthropic/claude-sonnet-5`, `google/gemini-3.5-flash` |
| **Ollama** | `ollama` | `ollama` | `llama4`, `qwen3.5`, `deepseek-v4` |

## 設定檔

CoreLayer 採用**設定與金鑰分離**的安全設計。設定檔保存在 `~/.jarvis/config/` 目錄下。

### 1. 基礎設定 (`config.json`)

定義啟用的供應商、預設模型和各個供應商的端點，此檔案**不包含任何 API 金鑰**：

```json
{
  "version": 1,
  "activeProvider": "mimo",
  "activeModel": "mimo-v2.5-pro",
  "providers": [
    {
      "id": "mimo",
      "name": "MiMo",
      "type": "openai_compatible",
      "baseURL": "https://token-plan-ams.xiaomimimo.com/v1",
      "enabled": true
    },
    {
      "id": "groq",
      "name": "Groq",
      "type": "openai_compatible",
      "baseURL": "https://api.groq.com/openai/v1",
      "enabled": true
    },
    {
      "id": "openrouter",
      "name": "OpenRouter",
      "type": "openai_compatible",
      "baseURL": "https://openrouter.ai/api/v1",
      "enabled": true
    },
    {
      "id": "ollama",
      "name": "Ollama",
      "type": "ollama",
      "baseURL": "http://localhost:11434/v1",
      "enabled": true
    }
  ]
}
```

### 2. 金鑰設定 (`credentials.json`)

獨立保存金鑰，與 `config.json` 中的 `id` 一一對應。

```json
{
  "mimo": "your_mimo_api_key_here",
  "groq": "your_groq_api_key_here",
  "openrouter": "your_openrouter_api_key_here"
}
```

## 模型路由

當設定多個供應商時，模型閘道根據以下條件選擇：

- **任務複雜度** — 複雜推理路由到更強大的模型
- **延遲需求** — 時間敏感的請求偏好 Groq 或本地模型
- **成本最佳化** — 日常任務使用較便宜的供應商
- **可用性** — 當供應商故障時的備用方案

## 環境變數

你可以使用環境變數而不是硬編碼 API 金鑰：

```bash
GROQ_API_KEY=your_groq_api_key_here
OPENROUTER_API_KEY=your_openrouter_api_key_here
```

在設定中引用它們：

```json
{
  "apiKey": "${GROQ_API_KEY}"
}
```
