---
title: MCP 服务器配置
description: 配置模型上下文协议服务器连接。
---

CoreLayer 通过 MCP 服务器连接外部工具和数据源。每个服务器暴露的工具都可以在对话中被 Jarvis 调用，并受权限守卫管控。

配置文件位于 `~/.jarvis/config/mcp-servers.json`。配置格式如下：

```json
{
  "version": 1,
  "servers": [
    {
      "id": "filesystem",
      "name": "filesystem",
      "transport": "stdio",
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-filesystem", "/workspace"],
      "enabled": true,
      "permissions": {
        "read": true,
        "write": true,
        "delete": false,
        "bulkWrite": false
      },
      "riskPolicy": {
        "low": "auto",
        "medium": "notify",
        "high": "confirm",
        "critical": "deny"
      }
    },
    {
      "id": "github",
      "name": "github",
      "transport": "stdio",
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-github"],
      "env": {
        "GITHUB_TOKEN": "${GITHUB_TOKEN}"
      },
      "enabled": true,
      "permissions": {
        "read": true,
        "write": true,
        "delete": false,
        "bulkWrite": false
      },
      "riskPolicy": {
        "low": "auto",
        "medium": "notify",
        "high": "confirm",
        "critical": "deny"
      }
    }
  ]
}
```

## 服务器类型

### Stdio 服务器

大多数 MCP 服务器使用 stdio（标准输入/输出）进行通信：

```json
{
  "name": "my-server",
  "command": "node",
  "args": ["./my-mcp-server.js"]
}
```

### SSE 服务器

使用服务器发送事件的远程服务器：

```json
{
  "name": "remote-server",
  "url": "https://my-server.example.com/mcp"
}
```

## 管理服务器

### 通过控制中心

- **添加** — 设置 → MCP → 添加服务器
- **移除** — 点击服务器卡片上的删除按钮
- **重启** — 点击重启按钮重新连接
- **查看工具** — 查看已连接服务器暴露的所有工具

### 连接状态

| 状态 | 含义 |
|------|------|
| 已连接 | 服务器正在运行，工具可用 |
| 连接中 | 服务器正在启动 |
| 已断开 | 服务器已停止或崩溃 |
| 错误 | 服务器启动失败 — 检查日志 |

## 工具发现

当 MCP 服务器连接时，CoreLayer 会自动：

1. 发现服务器暴露的所有工具
2. 将它们注册到工具注册中心
3. 使它们可供 Jarvis 进行工具调用

来自 MCP 服务器的工具与原生工具受到相同对待 — 它们在执行前都要经过权限守卫。
