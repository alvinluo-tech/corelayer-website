El Model Gateway de CoreLayer enruta solicitudes a traves de multiples proveedores de IA. Puedes configurar uno o mas proveedores y dejar que Jarvis seleccione el mejor modelo para cada tarea.

## Archivos de ConfiguraciĂłn

CoreLayer separa la configuraciĂłn de las credenciales secretas. Los archivos de configuraciĂłn se guardan en el directorio `~/.jarvis/config/`.

### 1. ConfiguraciĂłn Principal (`config.json`)

Define los proveedores activos, las reglas de enrutamiento y los endpoints. **Este archivo no contiene claves API**:

```json
{
  "version": 1,
  "activeProvider": "mimo",
  "activeModel": "mimo-v2.5-pro",
  "providers": [
    {
      "id": "mimo",
      "name": "MiMo",
      "type": "openai_compatible",
      "baseURL": "https://token-plan-ams.xiaomimimo.com/v1",
      "enabled": true
    },
    {
      "id": "groq",
      "name": "Groq",
      "type": "openai_compatible",
      "baseURL": "https://api.groq.com/openai/v1",
      "enabled": true
    },
    {
      "id": "openrouter",
      "name": "OpenRouter",
      "type": "openai_compatible",
      "baseURL": "https://openrouter.ai/api/v1",
      "enabled": true
    },
    {
      "id": "ollama",
      "name": "Ollama",
      "type": "ollama",
      "baseURL": "http://localhost:11434/v1",
      "enabled": true
    }
  ]
}
```

### 2. ConfiguraciĂłn de Credenciales (`credentials.json`)

Guarda las claves API de forma segura, mapeando los valores al `id` del proveedor correspondiente:

```json
{
  "mimo": "your_mimo_api_key_here",
  "groq": "your_groq_api_key_here",
  "openrouter": "your_openrouter_api_key_here"
}
```

## Enrutamiento de Modelos

Cuando hay multiples proveedores configurados, el Model Gateway selecciona segun:

- **Complejidad de la tarea** â€” el razonamiento complejo se enruta a modelos mas capaces
- **Requisitos de latencia** â€” las solicitudes sensibles al tiempo prefieren Groq o modelos locales
- **Optimizacion de costes** â€” las tareas rutinarias usan proveedores mas economicos
- **Disponibilidad** â€” respaldo si un proveedor no esta disponible

## Variables de Entorno

Puedes usar variables de entorno en lugar de codificar las claves API:

```bash
GROQ_API_KEY=your_groq_api_key_here
OPENROUTER_API_KEY=your_openrouter_api_key_here
```

Referencialas en la configuracion:

```json
{
  "apiKey": "${GROQ_API_KEY}"
}
```
