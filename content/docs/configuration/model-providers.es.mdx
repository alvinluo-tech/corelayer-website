---
title: Proveedores de Modelos
description: Configura proveedores de modelos de IA para el enrutamiento de Jarvis.
---

El Model Gateway de CoreLayer enruta solicitudes a traves de multiples proveedores de IA. Puedes configurar uno o mas proveedores y dejar que Jarvis seleccione el mejor modelo para cada tarea.

## Proveedores Admitidos

El Model Gateway admite de forma nativa los siguientes proveedores a través de ajustes preestablecidos incorporados:

| Proveedor | ID | Tipo | Modelos Populares |
|-----------|----|------|-------------------|
| **MiMo (Xiaomi)** | `mimo` | `openai_compatible` | `mimo-v2.5-pro`, `mimo-2.5` |
| **Groq** | `groq` | `openai_compatible` | `llama-3.3-70b-versatile`, `mixtral-8x7b-32768` |
| **DeepSeek** | `deepseek` | `openai_compatible` | `deepseek-chat`, `deepseek-reasoner` |
| **Anthropic** | `anthropic` | `openai_compatible` | `claude-sonnet-4`, `claude-haiku-4-5` |
| **Google Gemini** | `google` | `openai_compatible` | `gemini-2.5-pro`, `gemini-2.5-flash` |
| **OpenAI** | `openai` | `openai_compatible` | `gpt-4o`, `gpt-4o-mini`, `o3-mini` |
| **Qwen (Alibaba)** | `qwen` | `openai_compatible` | `qwen-max`, `qwen-plus` |
| **OpenRouter** | `openrouter` | `openai_compatible` | `anthropic/claude-sonnet-4`, `google/gemini-2.5-pro` |
| **Ollama** | `ollama` | `ollama` | `llama3.2`, `qwen2.5`, `deepseek-r1` |

## Archivos de Configuración

CoreLayer separa la configuración de las credenciales secretas. Los archivos de configuración se guardan en el directorio `~/.jarvis/config/`.

### 1. Configuración Principal (`config.json`)

Define los proveedores activos, las reglas de enrutamiento y los endpoints. **Este archivo no contiene claves API**:

```json
{
  "version": 1,
  "activeProvider": "mimo",
  "activeModel": "mimo-v2.5-pro",
  "providers": [
    {
      "id": "mimo",
      "name": "MiMo",
      "type": "openai_compatible",
      "baseURL": "https://token-plan-ams.xiaomimimo.com/v1",
      "enabled": true
    },
    {
      "id": "groq",
      "name": "Groq",
      "type": "openai_compatible",
      "baseURL": "https://api.groq.com/openai/v1",
      "enabled": true
    },
    {
      "id": "openrouter",
      "name": "OpenRouter",
      "type": "openai_compatible",
      "baseURL": "https://openrouter.ai/api/v1",
      "enabled": true
    },
    {
      "id": "ollama",
      "name": "Ollama",
      "type": "ollama",
      "baseURL": "http://localhost:11434/v1",
      "enabled": true
    }
  ]
}
```

### 2. Configuración de Credenciales (`credentials.json`)

Guarda las claves API de forma segura, mapeando los valores al `id` del proveedor correspondiente:

```json
{
  "mimo": "your_mimo_api_key_here",
  "groq": "your_groq_api_key_here",
  "openrouter": "your_openrouter_api_key_here"
}
```

## Enrutamiento de Modelos

Cuando hay multiples proveedores configurados, el Model Gateway selecciona segun:

- **Complejidad de la tarea** — el razonamiento complejo se enruta a modelos mas capaces
- **Requisitos de latencia** — las solicitudes sensibles al tiempo prefieren Groq o modelos locales
- **Optimizacion de costes** — las tareas rutinarias usan proveedores mas economicos
- **Disponibilidad** — respaldo si un proveedor no esta disponible

## Variables de Entorno

Puedes usar variables de entorno en lugar de codificar las claves API:

```bash
GROQ_API_KEY=your_groq_api_key_here
OPENROUTER_API_KEY=your_openrouter_api_key_here
```

Referencialas en la configuracion:

```json
{
  "apiKey": "${GROQ_API_KEY}"
}
```
