---
title: 权限守卫
description: 基于风险的执行控制，带有审批工作流和审计日志。
---

权限守卫是 CoreLayer 的安全系统。它按风险级别分类工具调用，暂停高风险操作等待用户审批，并维护完整的审计追踪。

## 为什么需要它

能够调用工具的 AI 代理需要边界。没有权限系统，代理可能会：

- 不经询问就删除文件
- 使用你的凭据进行外部 API 调用
- 修改你不想更改的数据

权限守卫使工具执行**可见、可控、可审计**。

## 风险分类

每个工具调用都被分类：

| 风险级别 | 示例 | 默认策略 |
|----------|------|----------|
| **读取** | 列出文件、读取数据、搜索 | 自动允许 |
| **写入** | 创建/修改文件、更新记录 | 确认 |
| **外部** | API 调用、网络请求 | 确认 |
| **破坏性** | 删除文件、删除表 | 确认 + 警告 |
| **凭据敏感** | 访问令牌、密码 | 除非明确允许否则阻止 |

## 审批流程

```text
工具调用请求
  → 风险分类
  → 策略检查（allow / confirm / deny）
  → 如果需要确认：暂停，显示审批面板
  → 用户批准或拒绝
  → 执行或跳过
  → 记录决策到审计日志
```

## 审批面板

当需要审批时，控制中心显示：

- **工具名称** 和来源
- 正在传递的 **参数**
- **风险级别** 指示器
- 将会发生什么的 **说明**
- **批准** / **拒绝** / **始终批准** 按钮

## 审计日志

每次工具执行都记录以下信息：

- 时间戳
- 工具名称和来源
- 参数（已脱敏）
- 风险级别
- 用户决策（已批准/已拒绝/自动允许）
- 执行结果

在**控制中心 → 权限 → 审计日志**中查看日志。
