---
title: 技能
description: 扩展 Jarvis 能力的自定义脚本化工作流。
---

技能是扩展 Jarvis 能力的可复用工作流。它们将多个步骤 — 工具调用、条件逻辑和结构化输出 — 组合成命名操作，Jarvis 可以自动或按请求调用。

## 什么是技能？

技能是一个 `SKILL.md` 文件，定义了多步骤工作流。与单个工具调用不同，技能可以：

- 链接多个工具
- 包含条件逻辑
- 跨步骤维护状态
- 以结构化格式呈现结果

## 使用技能

Jarvis 会自动发现可用的技能，并在适当时调用它们。你也可以明确请求一个技能：

```text
对当前更改运行代码审查技能
```

## 技能定义格式

每个技能是一个带有结构化章节的 Markdown 文件（`SKILL.md`）。这是一个最小示例：

```markdown
# 我的技能

此技能功能的简短描述。

## 此技能的功能

为执行此技能的 AI 代理提供的更详细说明。

## 步骤

### 步骤 1：收集上下文
读取当前项目状态并识别相关文件。

### 步骤 2：执行操作
使用收集的上下文调用适当的工具。

### 步骤 3：报告结果
总结完成的工作和后续操作。
```

技能存放在 `skills/` 目录中。每个技能有自己的文件夹：

```text
skills/
└── corelayer-mcp-adapter/
    └── SKILL.md
```

## 技能来源

技能可以来自：

- **内置** — 随 CoreLayer 一起发布（例如 `corelayer-mcp-adapter`）
- **社区** — 由其他用户分享
- **自定义** — 由你编写

### 内置技能

| 技能 | 用途 |
|------|------|
| `corelayer-mcp-adapter` | 引导将任何项目适配为 CoreLayer 兼容的 MCP 服务器 |

## 编写自定义技能

1. 在 `skills/` 下创建文件夹：`skills/my-skill/`
2. 添加 `SKILL.md` 文件，定义你的工作流
3. 重启 CoreLayer — Jarvis 会自动发现新技能

`corelayer-mcp-adapter` 技能是一个很好的参考，展示了如何用清晰的说明、检查清单和示例来构建多步骤工作流。
