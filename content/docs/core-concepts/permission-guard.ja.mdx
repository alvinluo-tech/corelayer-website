---
title: パーミッションガード
description: 承認ワークフローと監査ログを備えたリスクベースの実行制御。
---

パーミッションガードはCoreLayerの安全システムです。ツール呼び出しをリスクレベル別に分類し、リスクのある操作をユーザー承認のために一時停止し、完全な監査証跡を維持します。

## なぜ存在するのか

ツールを呼び出せるAIエージェントには境界が必要です。権限システムがなければ、エージェントは以下を行う可能性があります:

- 確認なしにファイルを削除
- あなたの資格情報で外部API呼び出しを行う
- 変更を意図しなかったデータを変更する

パーミッションガードはツール実行を**可視、制御可能、監査可能**にします。

## リスク分類

すべてのツール呼び出しは分類されます:

| リスクレベル | 例 | デフォルトポリシー |
|------------|----------|----------------|
| **Read** | ファイル一覧、データ読み取り、検索 | 自動許可 |
| **Write** | ファイル作成/変更、レコード更新 | 確認 |
| **External** | API呼び出し、Webリクエスト | 確認 |
| **Destructive** | ファイル削除、テーブルドロップ | 確認＋警告 |
| **Credential-sensitive** | アクセストークン、パスワード | 明示的許可なしでブロック |

## 承認フロー

```text
ツール呼び出し要求
  → リスク分類
  → ポリシーチェック（allow / confirm / deny）
  → confirmの場合: 一時停止、承認パネル表示
  → ユーザーが承認または拒否
  → 実行またはスキップ
  → 決定を監査証跡にログ記録
```

## 承認パネル

承認が必要な場合、コントロールセンターに以下が表示:

- **ツール名**とソース
- 渡される**パラメータ**
- **リスクレベル**インジケーター
- 何が起きるかの**説明**
- **承認** / **拒否** / **常に承認**ボタン

## 監査ログ

すべてのツール実行が以下とともにログに記録されます:

- タイムスタンプ
- ツール名とソース
- パラメータ（サニタイズ済み）
- リスクレベル
- ユーザーの決定（承認/拒否/自動許可）
- 実行結果

**コントロールセンター → 権限 → 監査ログ**でログを表示。
