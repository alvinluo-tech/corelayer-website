デーモンはバックグラウンドで実行されるNode.jsプロセスで、すべてのAIオーケストレーション、ツール実行、データ管理を処理します。

## アーキテクチャ

```text
Tauriデスクトップアプリ（Rust）
  └─ デーモンスーパーバイザー（Rust）
     └─ Node.jsデーモン（Hono HTTPサーバー）
        ├─ モデルゲートウェイ
        ├─ ツールレジストリ
        ├─ パーミッションガード
        ├─ MCPクライアントマネージャー
        └─ データベース（SQLite / Supabase / PostgreSQL）
```

## デーモンスーパーバイザー

TauriアプリにはRustベースのデーモンスーパーバイザーが含まれており、以下を行います:

- アプリ起動時にデーモンプロセスを**開始**
- デーモンのヘルスを**監視**
- デーモンがクラッシュした場合に**再起動**
- アプリ終了時にデーモンを**停止**

## HTTP API

デーモンは`localhost:3001`でHTTP APIを公開します（`config.json`の`daemon.port`フィールドで設定可能）。TauriフロントエンドはTauriのIPCブリッジを通じてこのAPIと通信します。

## 開発モード

開発中、デーモンはホットリロードで実行されます:

```bash
pnpm dev:daemon
```

`tsx watch`を使用してファイル変更時に再起動します。

## プロダクションモード

プロダクションビルドでは、デーモンはサイドカーバイナリとしてパッケージされます:

```bash
pnpm build:daemon:sidecar
```

サイドカーはTauriアプリにバンドルされ、別プロセスとして実行されます。

## ヘルス監視

コントロールセンターにデーモンステータスが表示されます:

- **Healthy** — デーモンが稼働中で応答あり
- **Starting** — デーモンが初期化中
- **Unhealthy** — デーモンが応答なし
- **Stopped** — デーモンが意図的に停止された
