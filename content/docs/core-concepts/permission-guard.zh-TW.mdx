權限守護是 CoreLayer 的安全系統。它按風險等級對工具呼叫進行分類，暫停有風險的操作以等待使用者核准，並維護完整的稽核軌跡。

## 為什麼需要它

能夠呼叫工具的 AI 代理需要邊界。沒有權限系統，代理可能會：

- 不詢問就刪除檔案
- 用你的憑證進行外部 API 呼叫
- 修改你無意更改的資料

權限守護使工具執行**可見、可控和可稽核**。

## 風險分類

每個工具呼叫都會被分類：

| 風險等級 | 範例 | 預設策略 |
|------------|----------|----------------|
| **讀取** | 列出檔案、讀取資料、搜尋 | 自動允許 |
| **寫入** | 建立/修改檔案、更新記錄 | 確認 |
| **外部** | API 呼叫、網頁請求 | 確認 |
| **破壞性** | 刪除檔案、清除資料表 | 確認 + 警告 |
| **憑證敏感** | 存取權杖、密碼 | 除非明確允許否則封鎖 |

## 核准流程

```text
工具呼叫請求
  → 風險分類
  → 策略檢查（允許 / 確認 / 拒絕）
  → 如需確認：暫停，顯示核准面板
  → 使用者核准或拒絕
  → 執行或跳過
  → 將決定記錄到稽核軌跡
```

## 核准面板

需要核准時，控制中心會顯示：

- **工具名稱**和來源
- 傳入的**參數**
- **風險等級**指標
- 將會發生什麼的**說明**
- **核准** / **拒絕** / **一律核准**按鈕

## 稽核日誌

每次工具執行都會記錄：

- 時間戳記
- 工具名稱和來源
- 參數（已清理）
- 風險等級
- 使用者決定（核准/拒絕/自動允許）
- 執行結果

在 **控制中心 → 權限 → 稽核日誌** 中查看日誌。
