モデルゲートウェイはAIモデルプロバイダーへの接続を管理し、利用可能な最適なモデルにリクエストをルーティングします。

## アーキテクチャ

```text
Jarvisリクエスト
  → モデルゲートウェイ
  → プロバイダー選択ロジック
  → Vercel AI SDK
  → プロバイダーAPI（Groq / OpenRouter / Ollama / OpenAI互換）
  → レスポンスがJarvisにストリーミングで返却
```

## Vercel AI SDK上に構築

モデルゲートウェイは[Vercel AI SDK v6+](https://sdk.vercel.ai)を使用:

- プロバイダー間の統一インターフェース
- ストリーミングレスポンス
- ツール呼び出しサポート
- 構造化出力

## ルーティングロジック

ゲートウェイは以下に基づいてモデルを選択:

1. **タスクタイプ** — 推論、コード生成、要約、分類
2. **レイテンシー要件** — リアルタイム音声には高速モデルが必要
3. **コスト** — 日常タスクには安価なモデルを使用
4. **フォールバック** — プライマリプロバイダーが利用不可の場合

## プロバイダー設定

設定詳細は[モデルプロバイダー](/docs/configuration/model-providers)を参照。

## ストリーミング

すべてのモデルレスポンスはデフォルトでストリーミングされます。これにより:

- テキストが生成されると同時に表示
- TTSが文の完了と同時に読み上げ可能（文レベルストリーミング）
- ツール呼び出しは識別されると同時に実行
