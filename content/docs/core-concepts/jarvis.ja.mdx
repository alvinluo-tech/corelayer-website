title: Jarvis AIアシスタント
description: CoreLayer全体の作業を調整するAIアシスタントペルソナ。

JarvisはCoreLayer内で動作する組み込みAIアシスタントです。スタンドアロンのチャットボットではなく、モデルゲートウェイを通じてリクエストをルーティングし、ツールレジストリ経由でツールを発見し、パーミッションガードを通じて安全性を確保するコーディネーターです。

## Jarvisの仕組み

```text
ユーザー入力（音声、テキスト、ショートカット）
  → 意図の解釈
  → コンテキスト構築（会話履歴、メモリ、ツール）
  → モデルゲートウェイによるモデル選択
  → ツール呼び出しを含むレスポンス生成
  → ツール実行前の権限チェック
  → 結果の配信
```

## 機能

- **自然言語タスク管理** — タスクの作成、クエリ、更新
- **リーディングリスト管理** — AI搭載のレコメンデーション付き記事トラッキング
- **日次/週次レビュー** — パターン認識によるアクティビティの要約
- **ツール呼び出し** — ネイティブモジュール、MCPサーバー、スキル、RESTアダプターからのツール実行
- **音声インタラクション** — ウェイクワード、ASR、TTS、割り込みを含む完全な音声パイプライン

## メモリ

Jarvisは会話全体にわたって長期メモリを維持します。メモリは設定されたデータベース（デフォルトではSQLite）にローカル保存され、セッション間で永続化されます。

- **会話コンテキスト** — 現在のセッション履歴、アクティブな会話中にメモリに保持され、セッション終了時にデータベースに書き込み
- **ユーザー設定** — 学習されたパターン（例: 好ましいモデル、頻繁に使用するツール、レスポンススタイル）が蓄積
- **タスクコンテキスト** — アクティブなタスク、期限、ステータス — Jarvisは作業について尋ねられた際にこれを参照
- **ツールコンテキスト** — 最近使用したツールとその結果、以前のアクションについて再クエリなしでフォローアップ可能

メモリはユーザーごとにスコープされ、Supabaseクラウド同期を有効にしない限り、CoreLayerインスタンス間で共有されることはありません。

## マルチモデルルーティング

Jarvisは単一モデルに依存しません。モデルゲートウェイは、複雑さ、レイテンシー要件、可用性に基づいて、各リクエストに最適なモデルを選択します。
