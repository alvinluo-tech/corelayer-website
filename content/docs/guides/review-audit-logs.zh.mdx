CoreLayer 中的每次工具执行都会被记录。本指南展示如何查看和使用审计日志。

## 访问审计日志

打开**控制中心 → 权限 → 审计日志**。

## 日志条目格式

每个条目包含：

```text
[2026-06-15 14:32:01] tool-name
  来源: mcp | native | skill | rest
  风险: read | write | external | destructive
  策略: allow | confirm | deny
  决策: approved | denied | auto-allowed
  参数: { ... }
  结果: success | error
```

## 过滤日志

按以下条件过滤：

- **日期范围** — 特定时间段
- **工具名称** — 查找特定工具调用
- **风险级别** — 仅查看写入/破坏性操作
- **决策** — 查找已拒绝或已批准的调用

## 常见用例

### 安全审查

检查进行了哪些外部 API 调用：

1. 按**风险：外部**过滤
2. 查看工具名称和参数
3. 验证所有调用都是预期的

### 调试

查找工具调用失败的原因：

1. 按**结果：错误**过滤
2. 检查日志条目中的错误消息
3. 验证工具参数是否正确

### 合规

生成所有数据修改的报告：

1. 按**风险：写入**和**风险：破坏性**过滤
2. 导出日志条目
3. 审查所有数据更改

## 日志保留

审计日志存储在配置的数据库中，跨会话持久化。默认情况下，日志无限期保留。要限制磁盘使用，在 `config.json` 中设置 `permissions.auditRetentionDays` — 超过指定天数的条目在启动时会自动清理。
