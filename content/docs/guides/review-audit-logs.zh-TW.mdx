CoreLayer 中的每次工具執行都會被記錄。本指南展示如何檢視和使用稽核日誌。

## 存取稽核日誌

開啟 **控制中心 → 權限 → 稽核日誌**。

## 日誌條目格式

每個條目包含：

```text
[2026-06-15 14:32:01] tool-name
  來源：mcp | native | skill | rest
  風險：read | write | external | destructive
  策略：allow | confirm | deny
  決定：approved | denied | auto-allowed
  參數：{ ... }
  結果：success | error
```

## 篩選日誌

按以下條件篩選：

- **日期範圍** — 特定時間段
- **工具名稱** — 尋找特定工具呼叫
- **風險等級** — 僅查看寫入/破壞性操作
- **決定** — 尋找被拒絕或核准的呼叫

## 常見使用情境

### 安全檢視

檢查進行了哪些外部 API 呼叫：

1. 按**風險：外部**篩選
2. 檢視工具名稱和參數
3. 驗證所有呼叫都是預期的

### 除錯

找出工具呼叫失敗的原因：

1. 按**結果：錯誤**篩選
2. 檢查日誌條目中的錯誤訊息
3. 驗證工具參數是否正確

### 合規

生成所有資料修改的報告：

1. 按**風險：寫入**和**風險：破壞性**篩選
2. 匯出日誌條目
3. 檢視所有資料變更

## 日誌保留

稽核日誌儲存在本地設定的資料庫中，並在不同工作階段之間持久化。預設情況下，日誌會無限期保留。若要限制磁碟使用量，在 `config.json` 中設定 `permissions.auditRetentionDays` — 超過指定天數的條目會在啟動時自動清除。
