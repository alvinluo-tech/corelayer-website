---
title: 監査証跡の確認
description: パーミッションガードの監査証跡を理解して操作します。
---

CoreLayerのすべてのツール実行はログに記録されます。このガイドでは、監査ログの確認と操作方法を説明します。

## 監査ログへのアクセス

**コントロールセンター → 権限 → 監査ログ**を開く。

## ログエントリのフォーマット

各エントリには以下が含まれます:

```text
[2026-06-15 14:32:01] tool-name
  Source: mcp | native | skill | rest
  Risk: read | write | external | destructive
  Policy: allow | confirm | deny
  Decision: approved | denied | auto-allowed
  Parameters: { ... }
  Result: success | error
```

## ログのフィルタリング

以下でフィルタリング:

- **日付範囲** — 特定期間
- **ツール名** — 特定のツール呼び出しを検索
- **リスクレベル** — write/destructive操作のみ表示
- **決定** — 拒否または承認された呼び出しを検索

## 一般的なユースケース

### セキュリティレビュー

行われた外部API呼び出しを確認:

1. **Risk: external**でフィルタリング
2. ツール名とパラメータを確認
3. すべての呼び出しが想定通りであることを確認

### デバッグ

ツール呼び出しの失敗原因を検索:

1. **Result: error**でフィルタリング
2. ログエントリのエラーメッセージを確認
3. ツールパラメータが正しかったか確認

### コンプライアンス

すべてのデータ変更のレポートを生成:

1. **Risk: write**と**Risk: destructive**でフィルタリング
2. ログエントリをエクスポート
3. すべてのデータ変更を確認

## ログ保持

監査ログは設定されたデータベースにローカル保存され、セッション間で永続化されます。デフォルトでは、ログは無期限に保持されます。ディスク使用量を制限するには、`config.json`で`permissions.auditRetentionDays`を設定 — 指定日数より古いエントリは起動時に自動的に剪定されます。
