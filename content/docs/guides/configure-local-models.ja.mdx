Ollamaを使用してAIモデルをマシン上で完全に実行します。

## Ollamaをインストール

[ollama.com](https://ollama.com)からダウンロードしてインストール。

## モデルをプル

```bash
ollama pull llama3.2
```

CoreLayerで人気のモデル:

| モデル | サイズ | ユースケース |
|-------|------|----------|
| `llama3.2` | 2-3GB | 汎用、高速 |
| `llama3.3:70b` | 40GB | 複雑な推論 |
| `qwen2.5:14b` | 9GB | 速度と能力のバランス良好 |
| `codellama:13b` | 7GB | コード生成 |

## CoreLayerを設定

Ollamaをプロバイダーとして追加:

```json
{
  "models": {
    "providers": [
      {
        "name": "ollama",
        "type": "ollama",
        "endpoint": "http://localhost:11434",
        "defaultModel": "llama3.2"
      }
    ]
  }
}
```

またはコントロールセンター経由:

1. **設定 → モデル**を開く
2. **プロバイダー追加**をクリック
3. **Ollama**を選択
4. エンドポイントを`http://localhost:11434`に設定
5. モデルを選択

## 確認

Jarvisに質問して、ローカルモデルを使用して応答するか確認:

```text
どのモデルで動いていますか？
```

## パフォーマンスのヒント

- **GPUアクセラレーション**: Ollamaは利用可能な場合、自動的にGPUを使用
- **モデルサイズ**: 小さいモデルは応答が速いが能力は低い
- **量子化**: 量子化モデル（Q4、Q5）を使用するとハードウェアが限られている場合にパフォーマンスが向上
