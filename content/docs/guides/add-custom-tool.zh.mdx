---
title: 注册自定义工具
description: 在工具注册中注册你自己的工具。
---

你可以通过向工具注册添加自定义工具来扩展 Jarvis。

## 方式一：MCP 服务器

推荐的添加工具方式是通过 MCP 服务器。创建一个简单的 MCP 服务器来暴露你的工具：

```typescript
import { McpServer } from "@modelcontextprotocol/sdk/server";
import { StdioServerTransport } from "@modelcontextprotocol/sdk/server/stdio";

const server = new McpServer({
  name: "my-tools",
  version: "1.0.0",
});

server.tool(
  "my-tool",
  "Description of what this tool does",
  {
    param1: { type: "string", description: "First parameter" },
  },
  async ({ param1 }) => {
    // Tool implementation
    return {
      content: [{ type: "text", text: `Result: ${param1}` }],
    };
  }
);

const transport = new StdioServerTransport();
await server.connect(transport);
```

然后将其添加到 CoreLayer：

```json
{
  "name": "my-tools",
  "command": "node",
  "args": ["./my-mcp-server.js"]
}
```

## 方式二：REST 适配器

对于已经作为 REST API 可用的工具，你可以将它们注册为 REST 适配器，而无需将它们包装在 MCP 服务器中。

在 CoreLayer 配置中添加 REST 适配器条目：

```json
{
  "name": "weather-api",
  "type": "rest",
  "baseUrl": "https://api.weather.example.com/v1",
  "auth": {
    "type": "bearer",
    "token": "${WEATHER_API_KEY}"
  },
  "tools": [
    {
      "name": "get-forecast",
      "description": "Get the weather forecast for a location",
      "method": "GET",
      "path": "/forecast",
      "parameters": {
        "type": "object",
        "properties": {
          "location": {
            "type": "string",
            "description": "City name or coordinates"
          }
        },
        "required": ["location"]
      }
    }
  ]
}
```

REST 适配器会自动将每个工具条目转换为工具注册中的可调用工具。Jarvis 向配置的端点发送 HTTP 请求，并将响应作为工具结果返回。

> REST 适配器最适合你不想包装的现有 API。对于新工具，首选 MCP 服务器 — 它们支持流式传输、更丰富的错误处理，并且更容易在本地测试。

## 工具元数据

注册工具时，请提供：

- **名称** — 唯一、描述性的标识符
- **描述** — 工具功能的清晰说明
- **参数** — 输入的 JSON Schema
- **风险级别** — 帮助权限守卫分类工具

## 最佳实践

- 使用描述性名称：`filesystem:read` 而不是 `fs1`
- 编写清晰的描述 — Jarvis 使用这些来决定何时调用工具
- 严格定义参数模式 — 防止无效调用
- 设置适当的风险级别 — 不要将所有东西都标记为 `read`
