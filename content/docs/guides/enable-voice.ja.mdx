---
title: 音声インタラクションを有効化
description: Jarvisとの音声インタラクションをセットアップします。
---

このガイドでは、CoreLayerの音声パイプラインを有効にする手順を説明します。

## 前提条件

- CoreLayerがインストール済み
- アプリにマイクアクセスが許可されている

## ステップ1: 音声を有効化

**設定 → 音声**を開き、**音声パイプラインを有効化**をトグル。

## ステップ2: ウェイクワードを設定

音声でJarvisをアクティブにするウェイクワード検出を有効化:

```json
{
  "voice": {
    "wakeWord": {
      "enabled": true,
      "keyword": "jarvis"
    }
  }
}
```

ウェイクワードはPicovoice Porcupineを使用してローカルで実行 — クラウド処理は不要です。

## ステップ3: ASRプロバイダーを選択

| プロバイダー | セットアップ |
|----------|-------|
| **Web Speech API** | セットアップ不要 — すぐに使用可能 |
| **Groq Whisper** | 設定 → モデルでGroqAPIキーが必要 |

はじめにはWeb Speech APIを推奨。Groq Whisperはより高い精度を提供します。

## ステップ4: 音声をテスト

ウェイクワード（"Jarvis"）と言うか、コントロールセンターのマイクボタンをクリック。そして、リクエストを話します:

```text
今日のタスクは何ですか？
```

Jarvisは音声を処理し、レスポンスを生成し、TTSで読み上げます。

## 音声オーバーレイ

音声がアクティブな場合、フローティングオーバーレイが表示されます:

- **リスニング** — 波形アニメーション、シアンアクセント
- **処理中** — スピニングインジケーター、バイオレットアクセント
- **スピーキング** — 再生進捗、エメラルドアクセント

## バージイン割り込み

再度話しかけることで、Jarvisのレスポンス中に割り込めます。パイプラインは現在のレスポンスを停止し、新しい入力を処理します。

## トラブルシューティング

- **ウェイクワードが検出されない**: マイクの権限を確認し、バックグラウンドノイズを軽減
- **音声が認識されない**: より明瞭に話すか、Groq Whisperに切り替え
- **オーディオ出力がない**: システムオーディオとTTS設定を確認
