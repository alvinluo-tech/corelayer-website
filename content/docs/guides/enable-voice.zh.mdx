---
title: 启用语音交互
description: 设置与 Jarvis 的语音交互。
---

本指南将引导你在 CoreLayer 中启用语音管线。

## 前提条件

- CoreLayer 已安装
- 已授予应用麦克风访问权限

## 步骤 1：启用语音

打开**设置 → 语音**，开启**启用语音管线**。

## 步骤 2：配置唤醒词

启用唤醒词检测，通过语音激活 Jarvis：

```json
{
  "voice": {
    "wakeWord": {
      "enabled": true,
      "keyword": "jarvis"
    }
  }
}
```

唤醒词使用 Picovoice Porcupine 本地运行 — 无需云端处理。

## 步骤 3：选择 ASR 提供商

| 提供商 | 设置 |
|--------|------|
| **Web Speech API** | 无需设置 — 开箱即用 |
| **Groq Whisper** | 需要在设置 → 模型中配置 Groq API 密钥 |

推荐从 Web Speech API 开始。Groq Whisper 提供更高的准确度。

## 步骤 4：测试语音

说出唤醒词（"Jarvis"）或点击控制中心的麦克风按钮。然后说出你的请求：

```text
我今天有什么任务？
```

Jarvis 将处理你的语音，生成响应，并使用 TTS 朗读出来。

## 语音覆盖层

当语音激活时，浮动覆盖层显示：

- **监听** — 波形动画，青色主题
- **处理** — 旋转指示器，紫色主题
- **朗读** — 播放进度，翡翠色主题

## 打断式交互

你可以通过再次说话来打断 Jarvis 的响应。管线将停止当前响应并处理你的新输入。

## 故障排除

- **唤醒词未检测到**：检查麦克风权限并减少背景噪音
- **语音未识别**：尝试说得更清楚或切换到 Groq Whisper
- **无音频输出**：检查系统音频和 TTS 配置
